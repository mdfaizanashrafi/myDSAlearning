## Introduction
Data Structures and Algorithms (DSA) form the foundation of efficient programming and problem-solving. Mastering DSA enables developers to optimize solutions, improve computational efficiency, and excel in technical interviews.

## Time and Space Complexity
Understanding **Big-O notation** is crucial in analyzing algorithm performance.
- **O(1)** - Constant Time
- **O(log N)** - Logarithmic Time
- **O(N)** - Linear Time
- **O(N log N)** - Log-Linear Time
- **O(N^2), O(2^N), O(N!)** - Quadratic, Exponential, and Factorial Time Complexity

## Data Structures
### 1. Arrays
- **Definition**: A collection of elements stored at contiguous memory locations.
- **Operations**: Traversal, Insertion, Deletion, Searching
- **Complexity**:
  - Access: O(1)
  - Search: O(N)
  - Insert/Delete: O(N)

### 2. Linked Lists
- **Types**: Singly, Doubly, Circular
- **Operations**: Insertion, Deletion, Traversal
- **Complexity**:
  - Access: O(N)
  - Search: O(N)
  - Insert/Delete: O(1) at head

### 3. Stacks
- **LIFO (Last In, First Out)** principle
- **Operations**: Push, Pop, Peek
- **Implementation**: Using Arrays or Linked Lists

### 4. Queues
- **FIFO (First In, First Out)** principle
- **Types**: Simple Queue, Circular Queue, Deque, Priority Queue

### 5. Hash Tables
- **Uses Hashing for fast access**
- **Operations**: Insert, Search, Delete in O(1) average time

### 6. Trees
- **Types**: Binary Trees, Binary Search Trees (BST), AVL Trees, B-Trees
- **Traversals**: Inorder, Preorder, Postorder, Level Order

### 7. Graphs
- **Types**: Directed, Undirected, Weighted, Unweighted
- **Representations**: Adjacency List, Adjacency Matrix
- **Traversal**: BFS (Breadth-First Search), DFS (Depth-First Search)

## Algorithms
### 1. Sorting Algorithms
- **Bubble Sort**: O(N^2)
- **Selection Sort**: O(N^2)
- **Insertion Sort**: O(N^2)
- **Merge Sort**: O(N log N)
- **Quick Sort**: O(N log N)

### 2. Searching Algorithms
- **Linear Search**: O(N)
- **Binary Search**: O(log N)

### 3. Recursion and Backtracking
- **Recursive Approach**: Base Case & Recursive Case
- **Backtracking**: Solving problems by exploring all possibilities

### 4. Dynamic Programming
- **Concept**: Breaking problems into subproblems and storing results to avoid recomputation
- **Examples**: Fibonacci, Knapsack, Longest Common Subsequence

### 5. Greedy Algorithms
- **Concept**: Making locally optimal choices to find global optimum
- **Examples**: Huffman Encoding, Dijkstra's Algorithm

### 6. Graph Algorithms
- **BFS & DFS**
- **Dijkstra's Algorithm (Shortest Path)**
- **Floyd-Warshall Algorithm (All-Pairs Shortest Path)**
- **Kruskal’s & Prim’s Algorithm (Minimum Spanning Tree)**

## Problem-Solving Approach
1. **Understand the problem statement**
2. **Identify input and output requirements**
3. **Choose the best data structure**
4. **Write a brute force solution**
5. **Optimize using efficient algorithms**
6. **Analyze time and space complexity**
7. **Implement and test**
